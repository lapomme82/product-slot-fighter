# 무협 슬롯 토너먼트 역기획서

작성 기준: 2026-05-20 현재 `main` 브랜치, 기능 기준 커밋 `4afd3fc Remove grand counter defense slot`
참고 자료: 현재 코드, `WORKLOG.md`, 최근 Git 히스토리, 배포 검증 내역

## 1. 프로젝트 개요

`무협 슬롯 토너먼트`는 플레이어가 출전 캐릭터를 구성하면 슬롯 릴 결과에 따라 자동 전투와 자동 토너먼트가 진행되는 2D 웹 게임이다.

핵심 경험은 다음과 같다.

- 플레이어는 캐릭터를 직접 조작하지 않고 엔트리 구성과 캐릭터 각성 선택에 집중한다.
- 전투는 슬롯 릴의 운, 캐릭터별 공격/방어 성향, HP/피해 수치에 의해 자동으로 결정된다.
- 결과는 즉시 계산되지만, 화면에서는 대진표, 슬롯 릴, 캐릭터 애니메이션, HP 변화, 로그를 통해 관전형으로 보여준다.
- 전체 톤은 사무실/무협/픽셀아트가 섞인 패러디형 자동 토너먼트다.

## 2. 플랫폼과 기술 스택

- 실행 플랫폼: 브라우저 기반 웹 게임
- 배포: GitHub Pages
- 프론트엔드: TypeScript, Vite
- 전투 렌더링: Phaser 3
- DOM UI: 타이틀, 캐릭터 선택, 대진표, 전투 로그
- 테스트: Vitest
- 주요 실행 명령:
  - `npm run dev`: 개발 서버 실행
  - `npm test`: 시뮬레이션 룰 테스트
  - `npm run build`: TypeScript 빌드 및 Vite 프로덕션 빌드
  - `npm run preview`: 로컬 프리뷰

## 3. 전체 게임 플로우

1. 타이틀 화면 진입
2. 토너먼트 버튼 선택
3. 캐릭터 선택 화면에서 출전 엔트리 구성
4. 필요 시 캐릭터를 각성 버전으로 전환
5. 엔트리 2명 이상 등록 시 게임 시작 가능
6. 등록 엔트리로 토너먼트 대진표 생성
7. 각 매치 자동 전투 진행
8. 전투 결과에 따라 다음 라운드 진출자 반영
9. 최종 우승자 결정

## 4. 화면 구성

### 4.1 타이틀 화면

목표는 설명 텍스트보다 이미지 중심의 첫인상을 만드는 것이다.

- 배경: `public/assets/stages/title-office-battle.png`
- 로고: `public/assets/ui/title-logo.png`
- CTA 버튼: `public/assets/ui/title-tournament-button.png`
- 화면 내 문구는 최소화되어 있으며, 기존 `WUXIA SLOT FIGHT`, 설명 문구, 상단 텍스트는 제거했다.
- 타이틀 화면에서는 상단 `타이틀` 버튼과 `일시정지` 버튼을 숨긴다.
- 버튼은 시스템 텍스트가 아니라 이미지 자산으로 표현한다.

### 4.2 캐릭터 선택 화면

역할은 엔트리 구성과 캐릭터별 기대감을 만드는 것이다.

- 기본 캐릭터 목록과 특별 참전 캐릭터 그룹을 보여준다.
- 캐릭터 카드를 선택하면 좌측 프로필 패널이 갱신된다.
- 엔트리 등록 시 우측 엔트리 목록에 추가된다.
- 이미 등록된 캐릭터는 `ENTRY` 배지가 표시된다.
- 각성 상태의 캐릭터는 `각성` 배지가 표시된다.
- 일반 캐릭터로 되돌리면 각성 배지는 제거된다.
- 엔트리에서 캐릭터를 제거하면 해당 캐릭터의 각성 상태도 함께 해제된다.

캐릭터 선택 프로필 연출:

- 일반/각성 모두 `기본 이미지 -> 소개 영상 -> 최종 이미지` 흐름을 사용한다.
- 기본 이미지에서 영상 첫 프레임으로 넘어갈 때는 전환 연출 없이 즉시 전환한다.
- 영상 종료 후 최종 이미지로 넘어갈 때만 페이드 아웃 후 즉시 최종 이미지가 표시된다.
- 이전에 있던 슬라이드성 위치 이동은 제거되어 기준점이 미세하게 흔들리지 않도록 했다.
- 일반/각성 전환은 카드가 뒤집히는 플립 연출로 표현한다.

### 4.3 대진표 화면

역할은 선택한 엔트리가 어떤 토너먼트 구조로 배치되었는지 보여주는 것이다.

- 최소 2명부터 시작 가능하다.
- 엔트리 수에 따라 2/4/8/16 슬롯 브래킷을 자동 선택한다.
- 빈 슬롯은 부전승으로 처리된다.
- 토너먼트 진행 중 완료된 매치는 결과가 반영되고 다음 라운드 진출자가 표시된다.
- 각성 캐릭터는 대진표와 엔트리 표시에도 각성 태그가 붙는다.

### 4.4 전투 화면

전투는 Phaser 씬에서 렌더링된다.

- 배경: `public/assets/stages/finance-center.png`
- 좌우 캐릭터, 상단 HUD, HP 바, 라운드/VS 표시가 있다.
- 전투 로그는 DOM 패널로 별도 표시된다.
- 전투 화면에서만 `일시정지` 버튼이 활성화된다.
- 전투 외 화면에서는 `일시정지` 버튼이 숨겨진다.

전투 시작 연출:

- 캐릭터 등장
- 아이들 애니메이션 시작
- 카운트다운
- 공격/방어 슬롯 릴 표시
- 턴별 슬롯 회전과 전투 모션 재생

전투 종료 연출:

- 슬롯 릴 페이드 아웃
- 패배자 사망 또는 퇴장 연출
- 승자 아이들 및 승리 바운스
- 승자 배너 표시
- 다음 매치 또는 최종 결과로 이동

## 5. 토너먼트 시스템

토너먼트는 시드 기반 난수로 재현 가능하게 구성된다.

- 엔트리 셔플 후 브래킷 슬롯에 배치한다.
- 엔트리 수보다 브래킷 슬롯이 많으면 부전승 슬롯을 만든다.
- 부전승 매치는 자동으로 승자를 확정한다.
- 실제 토너먼트 결과는 전투 리플레이 배열로 먼저 계산된다.
- 화면은 계산된 리플레이를 순차 재생한다.

브래킷 크기:

| 엔트리 수 | 브래킷 크기 | 시작 라운드 |
| --- | ---: | --- |
| 2 | 2 | 결승 |
| 3-4 | 4 | 준결승 |
| 5-8 | 8 | 8강 |
| 9-16 | 16 | 16강 |

현재 `applyHiddenCharacters()`는 타입과 반환 구조만 존재하며 실제 숨은 캐릭터 개입 이벤트는 비활성 상태다.

## 6. 전투 규칙

### 6.1 턴 구조

- 전투 시작 시 좌/우 중 선공을 랜덤 결정한다.
- 매 턴마다 공격자와 방어자가 번갈아 바뀐다.
- 공격자 슬롯 3개와 방어자 슬롯 3개를 굴린다.
- 3개 릴이 같은 심볼이면 해당 행동이 성공한다.
- 결과가 실패로 뽑힌 경우 3개 릴이 일부러 불일치하도록 만들어 `꽝`처럼 보이게 한다.
- 한쪽 HP가 0 이하가 되면 승자를 결정한다.
- 200턴 이상 승자가 없으면 남은 HP가 높은 캐릭터가 승리한다.

### 6.2 공격 슬롯

공격 슬롯 결과 후보:

| 결과 | 기본 가중치 | 효과 |
| --- | ---: | --- |
| 약공격 | 50 | 약공격 피해 적용 |
| 강공격 | 30 | 강공격 피해 적용 |
| 공격 실패 | 15 | 피해 없음 |
| 필살기 | 5 | 필살기 피해 적용 |

공격 심볼:

- 약공격: `public/assets/slot-ui/symbols/weak-attack.png`
- 강공격: `public/assets/slot-ui/symbols/strong-attack.png`
- 필살기: `public/assets/slot-ui/symbols/special.png`

### 6.3 방어 슬롯

방어 슬롯 결과 후보:

| 결과 | 기본 가중치 | 효과 |
| --- | ---: | --- |
| 방어 | 35 | 받는 피해 50% |
| 회피 | 25 | 강공격/필살기 회피, 약공격은 회피 실패 |
| 반격 | 20 | 약공격/강공격 피해를 공격자에게 반사, 필살기는 반격 실패 |
| 방어 실패 | 15 | 공격 피해를 그대로 받음 |

방어 심볼:

- 방어: `public/assets/slot-ui/symbols/block.png`
- 회피: `public/assets/slot-ui/symbols/dodge.png`
- 반격: `public/assets/slot-ui/symbols/counter.png`

최근 변경으로 `대반격`은 방어 슬롯 결과 후보, UI 라벨, 슬롯 순환, 슬롯 심볼 로딩에서 제거되었다. 기존 대반격 보정은 일반 반격 보정으로 이전했다.

### 6.4 피해 계산

- 공격 실패: 양측 피해 없음
- 방어 실패: 방어자가 공격 피해 전체를 받음
- 방어 성공: 방어자가 `ceil(공격 피해 * 0.5)`를 받음
- 회피:
  - 약공격 상대: 회피 실패, 방어자가 피해를 받음
  - 강공격/필살기 상대: 피해 없음
- 반격:
  - 약공격/강공격 상대: 방어자가 피해를 받지 않고 공격자가 피해를 되돌려 받음
  - 필살기 상대: 반격 실패, 방어자가 피해를 받음

## 8. 각성 시스템

각성은 캐릭터 선택 단계에서 캐릭터별 변형 상태를 선택하는 기능이다.

- 각성 가능한 캐릭터는 별도 이름, 칭호, 설명, 프로필 이미지, 소개 영상, 최종 이미지를 가진다.
- 각성 상태는 `awakenedFighterIds`로 관리한다.
- 전투 진입 시 Phaser registry에 각성 캐릭터 ID 목록을 전달한다.
- 전투 HUD와 포트레이트는 각성 상태를 반영한다.
- 각성 캐릭터용 sprite atlas가 존재하는 경우 전투 애니메이션도 각성 버전을 사용한다.
- atlas가 없는 캐릭터는 기존 프레임 이미지 기반 애니메이션으로 폴백한다.

각성 전환 UX:

- 일반과 각성 전환 시 카드와 프로필 패널에 플립 연출이 들어간다.
- 각성 표시 배지는 선택 카드, 엔트리, 대진표, 전투 HUD에 반영된다.
- 일반으로 되돌리면 카드의 각성 표시가 즉시 제거된다.

## 9. 전투 비주얼과 애니메이션

### 9.1 캐릭터 애니메이션

기본 모션 목록:

- weakAttack
- strongAttack
- special
- block
- dodge
- counter
- hurt
- grandCounter

현재 전투 룰에서는 `grandCounter`가 직접 호출되지 않는다. 대반격 삭제 이후 남아 있는 `grandCounter` 프레임/모션 자산은 레거시 정리 후보로 볼 수 있다.

애니메이션 시스템:

- 구형 방식: `public/assets/sprites/{fighterId}/{motion}/00.png` 형식의 6프레임 이미지
- 신형 방식: `public/assets/sprite-atlases/*.png` + `*.json` 메타데이터
- atlas 메타데이터는 프레임 위치, pivot, offset, duration, hitbox/hurtbox를 가진다.
- Phaser BootScene에서 이미지를 로드한 뒤 atlas 프레임을 수동 등록한다.

### 9.2 슬롯 UI

슬롯 UI는 기존 SVG 심볼/사각형 프레임에서 픽셀아트 PNG 프레임/심볼 기반으로 교체되었다.

- 공격 프레임: `public/assets/slot-ui/attack-slot-frame.png`
- 방어 프레임: `public/assets/slot-ui/defense-slot-frame.png`
- 심볼 아틀라스 원본은 2x3 구조였고, 현재는 개별 PNG로 분할되어 사용된다.
- 체크보드 배경은 투명 처리되었다.
- 공격자는 공격 프레임, 방어자는 방어 프레임을 사용한다.
- 슬롯 크기는 최초 적용 후 50% 축소되었다가, 그 축소 상태 기준 30% 확대된 현재 크기로 조정되었다.

현재 표시 기준:

- 프레임 폭: 179.4px 기준 스케일
- 심볼 표시 크기: 33.8px 기준
- 릴 간격: 약 44.85px
- 릴 애니메이션 이동폭도 현재 크기에 맞춰 축소/확대됨
